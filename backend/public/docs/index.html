Authentication Endpoints

7 endpoints

Endpoint

/auth/register

Request Body


{
  "email": "user@example.com",
  "password": "securePassword123",
  "first_name": "John",
  "last_name": "Doe",
  "phone": "+1234567890"
}

Response (201 Created)


{
  "id": "uuid",
  "email": "user@example.com",
  "first_name": "John",
  "last_name": "Doe",
  "message": "Registration successful. Please verify your email."
}

Required Headers

  • Content-Type application/json

Status Codes

  • 201 Created - Registration successful
  • 400 Bad Request - Invalid input
  • 409 Conflict - User already exists

Endpoint

/auth/login

Response (200 OK)


{
  "access_token": "jwt_token",
  "refresh_token": "refresh_jwt_token",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "first_name": "John",
    "role": "user"
  },
  "expires_in": 3600
}

Authentication Note

The access token from this endpoint should be included in the Authorization header for all protected endpoints as: Authorization: Bearer {access_token}

Error Handling

All endpoints return appropriate HTTP status codes and error messages. Below are the common error responses:

400 Bad Request 

{
  "error": "bad_request",
  "message": "Invalid request parameters",
  "details": {
    "email": "Invalid email format"
  }
}
401 Unauthorized 

{
  "error": "unauthorized",
  "message": "Invalid or missing authentication token"
}
404 Not Found 

{
  "error": "not_found",
  "message": "Resource not found"
}
500 Internal Server Error 

{
  "error": "internal_server_error",
  "message": "An unexpected error occurred"
}

Pagination

List endpoints support pagination with the following parameters:

Query Parameters

page

Page number (default: 1)

limit

Items per page (default: 10, max: 100)

Pagination Response Example


{
  "data": [
    // Array of items
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 150,
    "total_pages": 15
  }
}

Pagination Tips

  • • Always check the pagination object in the response
  • • Use total_pages to determine if more pages are available
  • • The limit parameter cannot exceed 100
  • • For performance, avoid requesting large pages unless necessary
